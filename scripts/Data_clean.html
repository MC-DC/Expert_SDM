Code overview

This document creates a presence-absence dataset for the Monarch butterfly (Danaus plexippus) from a complete-checklist citizen science dataset.

  • Presences: checklists where the Monarch was recorded
  • Absences: completed checklists in the same season where no Monarch was reported
  • Season filter: June, July, and August (core North American breeding season)

Filter to the breeding season

We retain only records from June (6), July (7), and August (8), which represent the core summer breeding window for Monarchs across North America.

breeding_months <- 6:8

breeding_checklists <- dat |>
  filter(month %in% breeding_months)

cat("Checklists in breeding season:", n_distinct(breeding_checklists$Checklist.ID), "\n")
## Checklists in breeding season: 59614

Identify Monarch presences

Extract all checklist IDs where Danaus plexippus was recorded.
Each unique checklist ID gets a presence = 1 flag.

monarch_present <- breeding_checklists |>
  dplyr::filter(Species_name == "Danaus plexippus") |>
  dplyr::select(Checklist.ID) |>
  distinct() |>
  mutate(presence = 1)

cat("Checklists with Monarch present:", nrow(monarch_present), "\n")
## Checklists with Monarch present: 8120

Build checklist-level metadata

Each checklist may contain many species rows, but spatial and temporal information is constant within a checklist. We collapse to one row per checklist using slice(1).

checklist_meta <- breeding_checklists |>
  group_by(Checklist.ID) |>
  slice(1) |>
  ungroup() |>
  dplyr::select(Checklist.ID, Site.ID, Latitude, Longitude,
         year, month, Sampling.protocol, Party.Size, Observer)

Create the presence-absence dataset

A left join assigns presence = 1 to checklists where the Monarch was seen. All unmatched checklists receive NA, which is then replaced with 0 (confirmed absence — the observer was present but did not record the species).

presence_absence <- checklist_meta |>
  left_join(monarch_present, by = "Checklist.ID") |>
  mutate(presence = replace_na(presence, 0))

Spatial thinning — one checklist per 1 km grid cell

To reduce spatial autocorrelation and sampling bias we overlay a 1 km × 1 km grid and keep a single record per cell. Where a cell contains both presences and absences, the presence is retained, this avoids introducing false absences in areas where the Monarch was genuinely detected.

The workflow is:

  1. Project coordinates to a metric CRS (UTM zone 18N, EPSG 32618)
  2. Assign each point to a grid cell using integer division on the projected coordinates.
  3. Within each cell, sort so that presences come first, then keep one row.

Create 1 km Hexagonal Grid over Study Area

hex_grid <- st_make_grid(

  cellsize    = 1000,

  square      = FALSE,
  flat_topped = TRUE

Spatially Join Occurrences to Grid

hex_sf <- st_sf(hex_id = seq_along(hex_grid), geometry = hex_grid)

# Join occurrences to hexagons
occ_hex <- st_join(occ_proj, hex_sf, join = st_within)

# Thin: keep 1 record per hexagon, prioritizing presences (presence == 1)
occ_thinned <- occ_hex %>%
  st_drop_geometry() %>%
  group_by(hex_id) %>%
  arrange(desc(presence), .by_group = TRUE) %>%  # presences (1) sort before absences (0)
  slice(1) %>%
  ungroup()

# Rejoin geometry from original data
occ_thinned <- occ_proj %>%
  filter(Checklist.ID %in% occ_thinned$Checklist.ID)
